# Plan: VitePress Documentation Site for iced-layout

## Context

iced-layout has a comprehensive XSD schema and parser covering 14+ widgets with rich attribute sets, but no documentation site. The goal is to generate a widget reference (one page per XML element) from the XSD schema, supplemented by hardcoded guide pages, and serve everything via VitePress on GitHub Pages.

## Implementation Steps

### 1. Workspace — `Cargo.toml`
Add `"docs-gen"` to the `members` array.

### 2. `docs-gen/Cargo.toml`
New binary package:
```toml
[package]
name = "docs-gen"
version = "0.0.1"
edition = "2024"

[[bin]]
name = "docs-gen"
path = "src/main.rs"

[dependencies]
quick-xml = "0.37"   # same version as xml/ crate — no new lock entries
serde = { version = "1", features = ["derive"] }
serde_json = "1"
```

### 3. `docs-gen/src/main.rs`

**Data structures:**
```rust
struct ElementDef { name, attributes: Vec<AttrDef>, child_count: ChildCount, has_text_content, has_state_children }
struct AttrDef { name, type_name, required: bool }
enum ChildCount { None, ZeroOrOne, ExactlyOne, ExactlyTwo, Unbounded, IfBranches }
```

**XSD parsing (2-pass streaming with `quick-xml`):**
- Pass 1 — collect `xs:simpleType` → `HashMap<name, Vec<enum_values>>`
- Pass 2 — collect `xs:element` → `Vec<ElementDef>` with attributes and child structure

**Key edge cases:**
- Strip `xs:` namespace prefix via `local_name()` helper
- `<if>` uses `xs:all` (not `xs:choice`) → `ChildCount::IfBranches`
- Style elements (`button-style`, etc.) have nested `<active>`/`<hovered>` children inside `xs:choice` — track depth, set `has_state_children = true`, skip attribute collection inside state children

**Hardcoded data (small tables in the binary):**
- `DESCRIPTIONS: &[(&str, &str)]` — 22 entries, one per element
- `EXAMPLES: &[(&str, &str)]` — minimal XML snippet per element

**Outputs generated by `main()`:**
1. `docs/widgets/{name}.md` for every element — via `generate_markdown()`
2. `docs/.vitepress/generated-sidebar.json` — via `generate_sidebar()`

Paths resolved via `CARGO_MANIFEST_DIR` parent = workspace root.

**Widget page Markdown template:**
```markdown
# `<{name}>`

{description}

## Example
```xml
{example}
```

## Attributes

| Attribute | Type | Required | Values |
|-----------|------|----------|--------|
| `attr`    | `T`  | Yes/No   | ...    |

## Structure

{structural_constraint_text}
```

Enum types are rendered as `"val1" | "val2" | "val3"` inline in the Values column.

For elements not yet implemented in the parser (toggler, text-editor, sensor): add a note "⚠️ Defined in the schema but not yet implemented in the parser."

**Sidebar JSON groups (hardcoded order in binary):**
- "Widgets": container, text, row, column, button, stack, space, text-input, vertical-slider, checkbox, tooltip, toggler, text-editor, sensor
- "Control Flow": if, foreach, widget
- "Styles": font, container-style, text-style, button-style, checkbox-style, text-input-style, toggler-style, text-editor-style

### 5. `docs/.vitepress/config.mts`
```typescript
import { defineConfig } from 'vitepress'
import generatedSidebar from './generated-sidebar.json'
import type { DefaultTheme } from 'vitepress'

export default defineConfig({
  title: 'iced-layout',
  description: 'XML layouts for the iced GUI framework',
  base: '/iced-layout/',
  themeConfig: {
    nav: [
      { text: 'Guide', link: '/guide/getting-started' },
      { text: 'Reference', link: '/widgets/container' },
    ],
    sidebar: {
      '/guide/': [{ text: 'Guide', items: [
        { text: 'Getting Started', link: '/guide/getting-started' },
        { text: 'XML Format', link: '/guide/xml-format' },
      ]}],
      '/widgets/': generatedSidebar as DefaultTheme.SidebarItem[],
    },
    socialLinks: [{ icon: 'github', url: 'https://github.com/...' }],
    search: { provider: 'local' },
  }
})
```

### 6. Hardcoded pages
- `docs/index.md` — VitePress `layout: home` with hero + features frontmatter
- `docs/guide/getting-started.md` — Cargo.toml dep, minimal XML file, minimal Rust usage of `layout!()`
- `docs/guide/xml-format.md` — `<layout>` root, attribute type reference table, event handler syntax, color format, interpolation syntax

### 8. `.github/workflows/docs.yml`
```yaml
on:
  push:
    branches: [main]
  workflow_dispatch:

jobs:
  build:
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@stable
      - run: cargo run -p docs-gen --release
      - uses: actions/setup-node@v4
        with: { node-version: '22', cache: 'npm', cache-dependency-path: docs/package-lock.json }
      - run: npm ci
        working-directory: docs
      - run: npm run docs:build
        working-directory: docs
      - uses: actions/configure-pages@v4
      - uses: actions/upload-pages-artifact@v3
        with: { path: docs/.vitepress/dist }
  deploy:
    needs: build
    environment: { name: github-pages }
    steps:
      - uses: actions/deploy-pages@v4
```
